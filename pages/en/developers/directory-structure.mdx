# Структура репозитория

Этот документ описывает файловую структуру [репозитория Mina] (https://github.com/minaprotocol/mina), и те роли, которые играют различные файлы:

- `dockerfiles/`  
   Содержит скрипты, связанные с Docker - TODO  объясняет это лучше

- `docs/`  
   Здесь вы можете ознакомиться с документами для участия в разработке кода и процессов. Сайт с документацией и пошаговыми инструкциями находится в `frontend/website/docs`.

- `frontend/`  
   Все коды, связанные с интерфейсами фронтенда и продуктами Mina

  - `wallet/`  
     Исходный код кошелька Mina

  - `website/`  
     Код для https://minaprotocol.com

    - `docs/`  
       Документация и инструкции по присоединению к сети Mina находятся по адресу https://minaprotocol.com/docs

    - `posts/`  
       Документы Markdown для постов в блогах

    - `src/`  
       Исходный код сайта

    - `static/`  
       Статические файлы, такие как изображения и т.д.

- `rfcs/`  
   Этот каталог содержит все принятые RFC (или "запросы на комментарии"), сделанные в соответствии с [RFC-процессом](https://github.com/MinaProtocol/mina/blob/master/CONTRIBUTING.md#rfcs).

- `scripts/`

- `src/`  
   Исходный код всех протоколов, включая и код приложения и библиотеки, находится в этом каталоге.

  - `*.opam`  
     Эти файлы необходимы для нашей системы сборки `dune`. Для каждой библиотеки в `lib` должно быть по одному файлу. 
     Когда вы создаете библиотеку `lib/foo_lib` с файлом `dune`, который задает имя библиотеки как `foo_lib`, вы должны создать файл `foo_lib.opam`.

  - `config/`  
     Конфигурация времени - эти .mlh-файлы определяют константы времени компиляции и их величины.

  - `app/`  
     Здесь все наши вэб-приложения.

    - `cli/`  
       Это клиент/daemon Mina. Это то, что вы используете для запуска стейкера, снаркера или простого клиента для отправки и получения транзакций.

    - `website/`  
       Каталог для веб-сайта скоро станет устаревшим - большая часть кода перенесена на `frontend/website/`. 

    - `reformat/`  
       Эта программа запускает `ocamlformat` на большинстве файлов в дереве исходников, за некоторыми исключениями.

    - `logproc/`  
       Эта утилита читает из `stdin`, с возможностью фильтрации и красивой печати сообщений журнала, генерируемых mina daemon.

    - `libp2p_helper/`  
       Эта программа использует go-libp2p для реализации peer-to-peer plumbing, который нужен для daemons Mina.

  - `external/`  
     Локальные копии внешних библиотек, в которые нам пришлось внести некоторые коррективы.

  - `lib/`  
     Библиотеки, продвигающие Mina.
    Библиотеки в основном делятся на две категории.
    1. Типы данных общего назначения и функционал. К ним относятся `snarky`, `fold_lib`, `vrf_lib`, `sgn` и другие.
    2. Специфический функционал приложения, структурированный как библиотека. Включает в себя `syncable_ledger`, `stage_ledger`, `transaction_snark` и другие.
